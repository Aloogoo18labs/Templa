About Templa

Templa is the web interface for the YangGo AI training registry. It supports:

  • Viewing training runs (dataset hash, config hash, model tier, epochs)
  • Registering new runs (coordinator must be whitelisted)
  • Attaching checkpoints and finalizing runs
  • Validator attestations and quorum tracking

This demo uses in-memory state. Connect to an EVM RPC and contract addresses for mainnet use.

YangGo contract: AI training software bot – datasets, model checkpoints, gradient attestations. Version 2.

Glossary

Run
A single training job: dataset hash, config hash, model tier, epoch count. Registered by a whitelisted coordinator.
Checkpoint
A snapshot hash attached to a run (e.g. model weights at an epoch).
Finalized
Run is locked; no more checkpoints; validators can attest.
Attestation
A validator’s vote (approved/rejected) on a finalized run.
Quorum
66% of validators have attested. Positive quorum: 66% of attestations are positive.
Model tier
1=base, 2=mid, 3=large, 4=xl. Affects fee multiplier.

FAQ

Who can register runs? Only addresses on the coordinator whitelist.

Who can attest? Any registered validator (min stake 0.1 ETH in production).

What is the fee? Base 0.01 ETH; tier 2–4 use multipliers 1.2x, 1.5x, 2x.

Where is data stored? In this demo, in browser memory. For production, use the YangGo contract on an EVM chain.

Integration notes

To connect to a deployed YangGo contract: set the contract address and RPC URL in your environment. Call registerRun, attachCheckpoint, finalizeRun (via coordinator), and attestRun (via validator) with the same parameters as in this UI. Use EIP-55 addresses and 40-hex hashes for dataset/config/checkpoint. Quorum is 66% (QUORUM_BPS 6600, BPS_DENOM 10000).

API surface (demo)

The global window.TemplaState object exposes: state, getRuns, getRunById, getGlobalStats, runExists, quorumReached, positiveQuorumReached, feeForTier, getTierName, attestationsNeededForQuorum, getRunTableData, getValidatorTableData, getDashboardSummary, exportAsJson, runDetailForModal, generateMockRuns, generateMockValidators, validateRunInput, validateAttestationInput. Use these for automation or testing.
